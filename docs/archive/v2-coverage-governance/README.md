## What This Is

This archive contains the previous product positioning, PRDs, and go-to-market strategy for Nolapse when the core narrative was:

> "The coverage floor that never moves"

This framing focused on general code quality governance and was used before the specific opportunity around AI agent scaling became clear.

---

## Why It's Archived

In March 2026, the product positioning evolved to address a specific, measurable growth constraint:

**v2.0:** *"Coverage governance for engineering teams"* (general, feature-focused)

**v3.0:** *"Quality infrastructure for AI agents"* (specific, growth-focused)

The v3 positioning directly addresses three growth blockers:
1. Agent scaling wall (manual review bottleneck)
2. Enterprise deal stall (AI code governance as procurement requirement)
3. Production incident risk (agent-written code quality)
